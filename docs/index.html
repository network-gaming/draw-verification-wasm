Independent Draw Verification

Every claim below is recomputed in your own browser from the published proof, using the same open-source algorithm as the platform — you can audit it, rebuild this verifier from it, or re-implement it in any language. The random value is fetched directly from the public drand network, not from this server — so you do not have to trust the platform's result.

How a verifiable draw works (30-second version)

Before the draw, the platform writes a tamper-evident commitment: a hash of its secret local seed, a digest of the frozen pool of eligible entries, and the number of a future drand beacon pulse — chosen before that pulse's random value exists. After the pulse publishes, it reveals the local seed, the beacon value, the final seed (SHA-256(local seed ‖ beacon value)) and the winners (a deterministic function of the pool and the final seed). Because the beacon value did not exist when the platform committed, it could not have known or steered the outcome — and anyone can reproduce the winners from the published values.

Paste the response of the platform's public proof feed (GET <platform>/v2/raffles/rounds/<id>/verification or /v2/raffles/draws/<id>/verification), or a saved bundle.
